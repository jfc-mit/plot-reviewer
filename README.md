# HEP Plot Reviewer

Automated feedback loop that reviews and polishes physics plots to publication quality.

## Architecture

```
Plot + Script
     |
     v
[1] Haiku spec check ──> executor fix if FAILs
     |
     v
[2] 3 Sonnets (one shot, parallel)
     - Physics referee: ACCEPT/REJECT, splits feedback into
       ANALYSIS ISSUES (upstream bugs) vs PLOT ISSUES (fixable)
     - Visual reviewer: checks rendered image against plotting-maxims.md
     - Expert physicist: freeform quality review, self-filters against spec
     |
     v
[3] Executor applies combined plot feedback (spec overrides reviewers)
     |
     v
[4] Haiku verifies with pytest-mpl style diff image
     - Was each feedback item addressed?
     - Any new breakage introduced?
     - If PASS -> done. If not -> one more fix, then stop.
```

Typical run: 2 iterations, $0.30-0.60, ~3-4 minutes.

## Files

```
feedback_loop.py          # Main loop — CLI and Python API
src/
  plotting-maxims.md      # Non-negotiable plotting standards (Visual + Code)
  haiku-check.md          # Full mechanical spec check prompt
  haiku-verify.md         # Post-fix verification prompt
  sonnet-physics.md       # Physics referee (splits analysis vs plot issues)
  sonnet-visual.md        # Visual reviewer (checks against maxims)
  sonnet-expert.md        # Expert physicist (evaluate -> read spec -> self-filter)
  executor-initial.md     # Initial plot generation prompt
  executor-fix.md         # Fix prompt (spec > reviewer, don't regress)
  plot_diff.py            # pytest-mpl style diff image generator
.claude/commands/
  hep-plotrev.md          # Interactive /hep-plotrev slash command
```

## Usage

### Python API (for pipelines like slopspec)

```python
from feedback_loop import review_plot

# Review an existing plot
result = review_plot(work_dir="/path/to/workdir")

# Review with physics context (helps the referee)
result = review_plot(
    work_dir="/path/to/workdir",
    task_context="Forward-backward asymmetry Q_FB vs cos(theta), Z->qq at 91.2 GeV",
)

# Generate + review from a task description
result = review_plot(
    work_dir="/path/to/workdir",
    task="Plot the Z mass peak from dilepton events, fit with Breit-Wigner",
)
```

### Return value

```python
{
    "passed": bool,           # Did verification pass?
    "physics_ok": bool,       # Did physics referee ACCEPT?
    "analysis_issues": str,   # Physics bugs for upstream (empty if none)
    "iterations": int,        # Number of executor fix rounds
    "cost": float,            # Total API cost in USD
    "versions": [Path, ...],  # All plot version snapshots
    "final_plot": Path,       # Path to final plot
    "log": str,               # Full log text
    "reviews": {              # Raw reviewer outputs
        "physics": str,
        "visual": str,
        "expert": str,
    },
}
```

### CLI

```bash
# Review existing plot
python feedback_loop.py --review-only /path/to/workdir

# Generate and review
python feedback_loop.py "Plot the Z mass peak..." /path/to/workdir
```

### Interactive (Claude Code)

```
/hep-plotrev /path/to/workdir --context "Z->ll template fit"
```

## Integration with slopspec

The key integration point is the `analysis_issues` field. When the physics
referee finds problems with the data itself (bad fit, wrong functional form,
suspicious values), these surface as analysis issues that the calling pipeline
needs to act on — they can't be fixed by editing the plot script.

```python
result = review_plot(
    work_dir=plot_dir,
    task_context=caption_text,  # from the analysis phase
)

# Route feedback
if result["analysis_issues"]:
    # Send back to analysis agent — physics bugs found in the plot
    analysis_agent.handle_feedback(result["analysis_issues"])

if not result["physics_ok"]:
    # MAJOR/REJECT — analysis needs rework before the plot is meaningful
    analysis_agent.rework(result["reviews"]["physics"])

# The plot itself is polished regardless (result["final_plot"])
```

The review loop runs as a separate subprocess (`claude -p`) per agent call,
so it's safe to run in parallel with other work as long as nothing else
writes to the same work directory.

## Work directory contract

The review loop expects:
- `make_plot.py` — the plotting script (will be edited by executor)
- `plot.png` — the rendered plot (will be regenerated by executor)

It produces:
- `plot_v0.png`, `plot_v1.png`, ... — version snapshots
- `diff_vN_vM.png` — pytest-mpl style diff images
- `plot.pdf` — PDF output (if executor follows spec)
- `loop_log.md` — full review transcript
